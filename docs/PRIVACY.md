# 개인정보 처리방침 (Privacy Policy)

이 문서는 `ev-mcp` (한국환경공단 EV 충전소 정보 MCP 서버) 가 사용자 데이터를 어떻게 다루는지 설명합니다. [Claude 원격 MCP 서버 디렉터리 제출 가이드](https://support.claude.com/ko/articles/12922490) 의 필수 문서입니다.

## 1. 개요

`ev-mcp` 는 한국환경공단의 **공공 OpenAPI(전기자동차 충전소 정보)** 를 Claude 가 사용할 수 있는 **원격 MCP 커넥터** 형태로 노출하는 무인증(no-auth) 프록시 서버입니다. 이 서버는 어떤 사용자 식별 정보도 보관하지 않으며, 처리하는 데이터는 모두 **공공 충전소 정보**(위치, 운영기관, 충전기 종류, 실시간 상태 등) 입니다. 사용자가 Claude 를 통해 도구를 호출하면, 서버는 받은 인자를 한국환경공단 API 로 그대로 중계하고 응답을 정형화하여 반환할 뿐입니다.

## 2. 수집하는 데이터

이 서버는 다음을 **수집·저장하지 않습니다**:

- 사용자 식별 정보(이름, 이메일, 전화번호, 사용자 ID 등)
- 로그인 자격 증명
- 쿠키, 세션, 광고 식별자
- 위치 데이터(클라이언트가 자발적으로 도구 인자로 보내는 좌표·주소 외에는 없음)
- 결제·과금 정보

서버 메모리에 일시적으로 보관되는 데이터는 **공공 데이터인 충전소 정보뿐**입니다. 구체적으로:

- `station_info` 캐시 — 충전소 정보 전체 목록 (TTL 24시간, 메모리 only)
- `status` 캐시 — 충전기 실시간 상태 (TTL 60초, 메모리 only)
- 정적 코드 테이블 — 시도/시군구/충전기종류 등 (저장소에 JSON 으로 동봉, 사용자 데이터 아님)

이 캐시들은 모두 **인메모리**(휘발성)이며, 컨테이너가 재시작하면 비워집니다. 디스크에 영구 기록되지 않습니다.

## 3. 외부로 전송되는 데이터

사용자가 Claude 에서 도구를 호출하면, 도구 인자(예: 좌표 `lat/lng`, 주소 문자열, 시도·시군구 코드, 충전소 ID 등) 가 다음 외부 API 로 전달됩니다:

| 외부 API | 도메인 | 운영 주체 | 역할 |
|---|---|---|---|
| 한국환경공단 EV 충전소 OpenAPI | `apis.data.go.kr/B552584/EvCharger` | 한국환경공단 (대한민국 정부 공공기관) | 충전소 정보·실시간 상태 조회 |
| VWorld 지오코더 (선택) | `api.vworld.kr` | 국토교통부 공간정보산업진흥원 (대한민국 정부) | 주소 → 좌표 변환 (`VWORLD_KEY` 가 설정된 경우에만) |

두 API 모두 **대한민국 정부에서 운영하는 공공 OpenAPI** 입니다. 각 기관의 자체 약관과 개인정보 처리방침을 따릅니다.

전송되는 인자는 사용자가 자연어 질문을 통해 자발적으로 Claude 에 입력한 것에서 추출됩니다. 본 서버는 그 인자를 가공하지 않고 외부 API 로 그대로 전달하며, 결과를 받은 후 응답에서 시크릿(인증키) 만 마스킹한 뒤 Claude 에 돌려줍니다.

## 4. 시크릿 처리

- 한국환경공단 OpenAPI 의 `SERVICE_KEY` 와 (선택) `VWORLD_KEY` 는 **서버 환경변수에만** 존재합니다.
- 시크릿 값은 **사용자에게 절대 노출되지 않습니다** — 도구 응답, 에러 메시지, 로그 어디에도 등장하지 않습니다.
- 모든 외부 API 응답·예외 메시지는 `EvChargerClient.redact()` 와 동등한 마스킹을 거치며, **raw 형태와 URL-encoded 변형(`urllib.parse.quote`, `quote_plus`) 까지 모두 마스킹**됩니다. (`.claude/rules/secrets.md` 참조)
- 시크릿은 코드·테스트·도커 이미지에 빌드타임으로 들어가지 않고, **런타임에 환경변수**로만 주입됩니다 (Render Dashboard 의 Environment 시크릿 또는 로컬 `.env`).
- `pydantic.SecretStr` 로 감싸 메모리에서도 우발적 출력(예: `repr()`) 으로 노출되지 않게 합니다.

## 5. 로그 보관

- 운영 환경(Render) 에서는 **stdout 으로 JSON 한 줄 로그**만 출력됩니다.
- 로그 라인에는 `event`, `level`, `logger`, `timestamp` 등 **운영 메타데이터만** 포함되며, **사용자 식별 정보는 일체 없습니다**.
- 로그는 Render 의 기본 로그 스트림에 보존되며, 서비스 운영 외 목적으로 사용되지 않습니다.
- 디스크에 영구 저장되는 별도 로그 파일은 없습니다(`access_log=False`, 헬스체크 조회 무로깅).
- 사용자가 GitHub Issue 등 이슈 트래커로 문의한 내용 외에는 **어떤 사용자 입력도 영구 저장하지 않습니다**.

## 6. 쿠키 / 트래킹 / 분석

- **쿠키 사용 없음.** 본 서버는 streamable HTTP MCP 엔드포인트 (`/mcp/`) 와 헬스체크 (`/health`) 만 제공하며, 브라우저 세션·쿠키를 사용하지 않습니다.
- **분석 도구 없음.** Google Analytics, Mixpanel, Amplitude 등 어떤 트래킹·분석 SDK 도 통합되어 있지 않습니다.
- **광고 없음.**

## 7. GDPR / 한국 개인정보보호법

- 본 서버는 **개인정보(personal data / 개인정보) 를 처리하지 않습니다.**
- 따라서 GDPR (EU 일반 개인정보 보호법) 의 "데이터 컨트롤러" 또는 "프로세서" 의무, 한국 「개인정보 보호법」 상의 개인정보처리자 의무가 적용될 대상 데이터가 존재하지 않습니다.
- 도구 인자로 전송되는 좌표·주소는 **사용자가 자발적으로 입력한 일회성 쿼리 파라미터** 로, 본 서버는 이를 저장하지 않으며 외부 공공 API 로 즉시 전달 후 폐기합니다.
- 그럼에도 불구하고, 사용자가 개인 식별 가능한 주소(예: 자택)를 자연어 질문에 포함시키면 한국환경공단·VWorld API 로 그 문자열이 전달될 수 있습니다. 이는 두 정부 API 의 처리방침에 따릅니다.

## 8. 데이터 삭제 요청

본 서버는 사용자 데이터를 보관하지 않으므로 별도의 삭제 요청 절차가 필요하지 않습니다.

- 캐시는 컨테이너 재시작 시 자동 폐기됩니다 (수동 삭제 불필요).
- 외부 정부 API 로 전송된 쿼리에 대한 처리는 해당 기관의 정책을 따릅니다.
- 그럼에도 문의·우려가 있다면 `docs/SUPPORT.md` 의 이슈 트래커로 연락하시면 best-effort 로 응답드립니다.
