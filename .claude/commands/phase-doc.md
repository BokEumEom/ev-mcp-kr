Phase **$ARGUMENTS** 결과를 `docs/PHASE$ARGUMENTS.md` 에 기록합니다.

## 템플릿

```markdown
# Phase $ARGUMENTS 보고 — {제목}

**기간:** {YYYY-MM-DD}
**범위:** {계획서의 Phase 섹션 한 줄 요약}
**검증:** pytest {N}건 통과, ruff 클린, mypy --strict 클린.

## 요약 (3줄)

{이번 Phase 가 무엇을 만들었나, 왜 만들었나, 다음 Phase 와 어떻게 연결되나}

| `src/ev_mcp/...` | ... |

## 핵심 설계 결정

- **결정 1:** 무엇 / 왜 / 트레이드오프
- **결정 2:** ...

리뷰에서 잡힌 CRITICAL/HIGH 와 처리:

|---|---|---|
| ... | ... | ... |

## 회귀 테스트 추가

- 무엇을 검증하는지 한 줄씩 (몇 번째 회귀인지)

## 변경 이력

- {YYYY-MM-DD HH:MM} 초안 작성

## 다음 단계 (Phase $((ARGUMENTS + 1)))

- {계획서의 다음 Phase 섹션 + 이번 Phase 가 남긴 미해결 항목}
```

## 작성 규칙

1. 위 템플릿을 그대로 복붙하지 말고, 이번 Phase 에 실제로 일어난 일로 채울 것.
2. "요약 (3줄)" 은 진짜 3줄. 길어지면 다른 섹션으로 옮겨.
3. "핵심 설계 결정" 에 의사결정 기록(가벼운 ADR)을 인라인으로. 별도 docs/ADR/ 디렉터리는 만들지 않음.
4. 회귀 테스트는 "테스트 N+1: 빈 응답 → 빈 list" 처럼 무엇을 막는지 한 줄.
5. 작성 후 `/verify` 한 번 더 돌려 그린 확인.
