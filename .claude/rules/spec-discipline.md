# Rule: 스펙 규율 (docx = 진실의 원천)

## 원칙

`/home/bokeum/ai/ev_mcp/한국환경공단_전기자동차 충전소 정보_OpenAPI활용가이드_v1.23.docx` 가 모든 모델·코드 테이블·필드 의미의 **유일한 진실의 원천**입니다.

- 모델 필드 추가/변경/삭제는 docx의 표를 직접 인용해 PR/커밋 메시지에 근거를 남길 것.
- 추측으로 필드를 만들지 말 것. 의문 생기면 docx를 다시 읽거나 `/spec-check` 슬래시 명령으로 감사.
- `src/ev_mcp/codes/*.json` 은 **사람이 직접 편집 X**. `scripts/extract_*.py` 만 통해 재생성.

## 새 docx 버전이 들어왔을 때

1. 파일명에서 버전 확인 (`..._v1.23.docx`).
2. `/spec-check` 실행 → 코드와의 차이 보고.
3. 변경 항목별로 todo 분리해 각각 한 커밋.
4. 모든 코드 테이블은 추출 스크립트로만 갱신.
5. `docs/PHASE{N}.md` 또는 별도 메모에 변경 이력 기록.

## 불일치 발견 시

- "docx 가 이렇게 적혀 있는데 코드는 이래요" 형식으로 사용자에게 보고.
- docx 가 우선이지만, 운영 응답이 docx와 다르게 동작하는 경우(현장 사례)는 별도 처리:
  - 모델은 docx대로 두되 BeforeValidator 로 graceful fallback (예: 미정의 stat 코드 → UNKNOWN).
  - 그 사유를 코드 주석 + 회귀 테스트로 박제.
